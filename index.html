HTML

  • The "frame" of the webpage comprised of elements enclosed in brackets, primarily the < head > and < body >
  • The < head > element contains information about the webpage.
      • - Can contain nested child elements including < title >, < style >, < link >, < meta >, < base >, and < script >
  • The < body > element represents the visible content show to the use.
      • - The body element is an example of a semantic HTML element that describes its content and role on a webpage (body, section, etc.).
      --- try to use when possible, as it makes code easier to read for the browser and other developers.
  • Some elements include attributes that further define what is shown or to add more information, such as < a href > when creating links or < section class > when creating sections
  • Can contain many nested child elements, including:
      • - headers: < h1-h6 > header of a section/paragraph. Use in order and only use h1 ONCE per page.
      - sections: < section > division of webpages into organized areas
      - paragraphs: < p > blocks of text or paragraph
      - unordered list & ordered lists: < ul > & < ol > used to create lists on webpage document
      - list items: < li > the individual items of a list (such as each bullet point on here!)
      - images: < img > the images/pictures used on the page
      - empty spaces between content: < br > to separate content as needed
      - anchors: < a > used to create links
  • Also can use a < footer > element, primarily used for information about author, copyright, contact, sitemap, and navigation.

CSS

  • 3 Ways to Style a Page
  • - inline CSS: style an element directly in the HTML file
  • - internal style sheet: embeds an entire stylesheet directly in the HTML file
  • - external CSS style sheet: a linked external file that holds selected CSS rules in a separate file; usually the preferred for "separation of concerns" coding

  • CSS rules generally have selectors (div, header, h1-h6, img) and declarations (property: value;) such as margin or padding, assigned with a value
  • * selector: special wild card symbol that says all elements on the page will have style properties assigned
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Can combine two selectors if both will have same styles applied (header and footer example in code)
  • Class Selector can be used to assign CSS styles to elements classed as same name in rule (ex. .card {width: 80%;} will apply style attributes to all HTML elements classed as "card")
  • Can also adjust width, height, border, box-shadow of elements in the HTML file

Git

  • Git: version control system
  • Github: cloud based hosting service to track projects and versions in repositories remotely
  • git status: checks what branch we are currently on

    • Create a Branch

  • git checkout -b branch-name: creates a new branch and switches to it
      • - git: preceding requirement for every Git statment
      - checkout: move working branch to a new branch
      - -b: flag that creates new branch
      - branch-name: name of the new branch; good practice to use prefix for what branch is intending to change

    Add Files and Commit changes:

  • git add -A
      • - add: git command to add files/changes
      - -A: flag used to for ALL changes
  • git commit -m "[description]": commits changes with a description of what was changed
      • - commit: command to commit staged content as a new commit snapshot
      - -m: flag to add a message
      - description: should be imperative and in present tense, such as "change font size"

    JavaScript

    • A variable is a named container that allows us to store data in our code.
    • Three keyword options for variable declaration:
        • - "const": hard sets the data of the variable
        - "let": allows for the variable data to possibly change if needed
        - "var": not commonly used in modern Javascript
    • Control flow is the order in which a computer executes code in a script.
    • Conditional logic (if and if/else statements) look for a "truthy" in order to execture
        • - A "falsy" (0, -0, 0n, "", null, undefined, NaN, or the Boolean value "false") will cause the statement to skip that condition
    • Arrays: a single variable that contains a group of data (including numbers, strings, Boolean value, and any combination of those value types); Arrays are zero indexed, which means the first index count beings at "0"
    • "For" loops allow for code to be repeatedly run and re-run as needed; 3 statements in a "For" loop:
        • - Starting point for the loop (ex: let x = 0)
        - Condition: what determines if a loop keeps running (ex: x < shapes.length)
        - What allows the array to reiterate over each item: increasing or decreasing in order to loop over desired array selections (ex: x++ (shorthand to incrementally increase by 1, or x + 1))
    • Functions: a set of instructions that can tell the computer what to do; 2 parts:
        • - Define a function: use the 'function' keyword, assign a function name followed by the () (ex: 'myName ()' ), and the code block containing the desired instructions with the {} (ex: {console.log("Jeremy")})
        - Call the function: running the function by using the function name (); for example, myName() after the function is defined (due to ßControl Flow)
