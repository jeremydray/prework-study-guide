HTML

  • The "frame" of the webpage comprised of elements enclosed in brackets, primarily the < head > and < body >
  • The < head > element contains information about the webpage.
      • - Can contain nested child elements including < title >, < style >, < link >, < meta >, < base >, and < script >
  • The < body > element represents the visible content show to the use.
      • - The body element is an example of a semantic HTML element that describes its content and role on a webpage (body, section, etc.).
      --- try to use when possible, as it makes code easier to read for the browser and other developers.
  • Some elements include attributes that further define what is shown or to add more information, such as < a href > when creating links or < section class > when creating sections
  • Can contain many nested child elements, including:
      • - headers: < h1-h6 > header of a section/paragraph. Use in order and only use h1 ONCE per page.
      - sections: < section > division of webpages into organized areas
      - paragraphs: < p > blocks of text or paragraph
      - unordered list & ordered lists: < ul > & < ol > used to create lists on webpage document
      - list items: < li > the individual items of a list (such as each bullet point on here!)
      - images: < img > the images/pictures used on the page
      - empty spaces between content: < br > to separate content as needed
      - anchors: < a > used to create links
  • Also can use a < footer > element, primarily used for information about author, copyright, contact, sitemap, and navigation.

Git

  • Git: version control system
  • Github: cloud based hosting service to track projects and versions in repositories remotely
  • git status: checks what branch we are currently on

    • Create a Branch

  • git checkout -b branch-name: creates a new branch and switches to it
      • - git: preceding requirement for every Git statment
      - checkout: move working branch to a new branch
      - -b: flag that creates new branch
      - branch-name: name of the new branch; good practice to use prefix for what branch is intending to change

    Add Files and Commit changes:

  • git add -A
      • - add: git command to add files/changes
      - -A: flag used to for ALL changes
  • git commit -m "[description]": commits changes with a description of what was changed
      • - commit: command to commit staged content as a new commit snapshot
      - -m: flag to add a message
      - description: should be imperative and in present tense, such as "change font size"
